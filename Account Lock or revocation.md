
# Symptoms

If your account has been blocked or your access has been erroneously revoked or your contract date was not updated you would notice that you get an error message like the following:

# Course of action

> The following steps are not all guaranteed to be needed as the revoking of access doesn't seem to be the same for everyone and every situation. 

Contacting help desk can help but they will follow a very similar path to what will be shown. As well in this case is extremely rare that your line manager can do anything since a line manager has limit access. So when helpdesk throws out a blanket statement such as:  *"Your line manager needs to re-instate your MyAccess profile"* or something like that, the line manager can do much other than really change details of your account and update your contract end date and status. 

# Steps

## 1. Create a remedy request re-enable your Microsoft active directory account(Your SBG account you use to login)

We need to make a remedy incident to reinstate your ad account. That is the root cause of the issue here. The details are as follows:

| Field           | Value                                               |
| :-------------- | :-------------------------------------------------- |
| Company         | Standard Bank of South Africa (Pty) Limited         |
| Customer        | (Person affected)                                   |
| Contact         | Whoever can be contacted or is helping              |
| Notes           | (YOUR_EMAIL) is disabled                            |
| Template        | Enterprise Password Vault (EPV)- Unable to access   |
| Summary         | Location & Department - Accounts - Unable to access |
| Service         | Security Services South Africa                      |
| CI              | MyAccess - Sailpoint                                |
| Impact          | 4-Minor/Localized                                   |
| Urgency         | 4-Low                                               |
| Priority        | Low                                                 |
| Incident Type   | User Service Request                                |
| Reported Source | Not too sure if it Matters can just put Phone       |

## 2. Escalate if needed

If your remedy request is not moving then try escalate it to teh best of your ability to get it moving

## 3. Office 365 license

You may need to then contact helpdesk or create a remedy incident to re-enable and assign a office 365 license back to your account to access stuff like teams.
